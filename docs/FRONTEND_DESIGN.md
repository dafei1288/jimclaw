# JimClaw 前端功能与设计文档 (FRONTEND_DESIGN.md)

## 1. 项目概述
JimClaw 前端是一个面向开发者的“自动化软件工程指挥中心”。它不仅展示多智能体协作的执行结果，更侧重于展示**执行过程的可观测性**（过程透明）和**人机协作的深度集成**（人工干预）。

## 2. 核心功能需求

### 2.1 实时任务流监控 (Live Command Center)
- **实时日志流**：通过 Socket.io 接收来自各 Agent 的 `agent-event`。
- **智能体标识**：为不同智能体（PM-观止、架构师-独孤、开发-星河、测试-清扬）分配独特的配色、图标和角色说明。
- **动作可视化**：
    - **思考 (Thinking)**：展示 Agent 的逻辑推导过程。
    - **工具调用 (Tool Use)**：点击可展开查看具体的 shell 命令输出或文件读写详情。
    - **LLM 洞察**：支持展开查看原始的 Prompt 和 LLM 响应内容。

### 2.2 决策与质量看板 (Dashboard)
- **执行阶段追踪 (Workflow Pipeline)**：
    - 阶段：需求分析 -> 架构设计 -> 契约同步 -> 任务拆解 -> 编码开发 -> 质量测试 -> 服务部署 -> 复盘归档。
    - 状态：未开始、进行中（脉冲动画）、已完成、失败。
- **项目共识池 (Consensus Pool)**：
    - 按类型（需求、技术、决策、问题、方案）分类展示团队达成的共识，确保跨节点信息一致性。
- **子任务追踪 (SubTask Tracker)**：
    - 实时同步 `state.subTasks` 的完成度，支持进度条展示和失败原因预览。
- **质量反馈 (QA & Mediation)**：
    - 聚合展示测试失败堆栈和架构师的仲裁修复指令。

### 2.3 文件资产浏览器 (Workspace Explorer)
- **实时文件树**：通过 `/api/workspace/files` 接口动态获取当前运行目录下的文件列表。
- **代码预览**：集成 `highlight.js`，支持 TS/JS/Py/JSON/HTML 等多种语言的语法高亮。
- **状态对齐**：支持点击刷新，观察 Agent 编写代码的动态过程。

### 2.4 人机协作 (Human-in-the-loop)
- **方案授权 (Approval)**：在启动编码前，强制弹出审批窗口，要求用户确认架构师的设计方案。
- **实时反馈 (Feedback)**：用户可随时通过“质疑”或“协助”功能，向当前运行的 Agent 发送文字指令。

## 3. 技术架构设计

### 3.1 前端技术栈
- **UI 框架**: TailwindCSS (Utility-first CSS)
- **图标库**: Lucide-static
- **实时通信**: Socket.io-client (v4.x)
- **代码高亮**: Highlight.js
- **渲染引擎**: 原生 JavaScript (DOM 操作优化)

### 3.2 数据流模型 (Event-Driven)
前端作为一个无状态的“观察者”，通过监听以下 Socket 事件驱动 UI 更新：
- `session-sync`: 页面初始化时的全量状态同步。
- `state-update`: 节点切换或进度变化时的增量状态更新。
- `agent-event`: 粒度最细的原子动作事件（包含 Tracing 信息）。
- `human-feedback-ack`: 确认用户反馈已被内核接收。

### 3.3 分布式追踪 (Tracing) 集成
- 每个 `agent-event` 携带 `traceId` 和 `spanId`。
- 前端在显示日志时，会自动计算 `durationMs`，展示每个动作的耗时。
- 顶部状态栏显示当前活跃的 `Node` 名称和 `北京时间`。

## 4. 交互设计规范
- **暗黑模式**: 默认采用 Slate-900 背景，减少长时间盯着屏幕的视觉疲劳。
- **响应式布局**: 侧边栏在移动端自动折叠，适配不同尺寸的显示终端。
- **状态着色**:
    - **运行中**: 靛蓝色 (Indigo) 脉冲效果。
    - **成功**: 翡翠绿 (Emerald) 静态显示。
    - **警告/重试**: 琥珀黄 (Amber)。
    - **错误/失败**: 玫瑰红 (Rose)。
